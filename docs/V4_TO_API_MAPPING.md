# Customs Calculator V4 → API Mapping (Delta from V3)

Документ фиксирует только отличия v4 от текущей реализации API и намеченные точки интеграции (за фича-флагом V4_CALC_ENABLED).

- **Фича-флаг**: `V4_CALC_ENABLED` (ENV, default=false). Используется в `api/calculator/services.py` внутри `CustomsCalculator.estimate()`.

- **EV (легковые)**
  - v4: пошлина = 15% от стоимости для всех аудиторий и возрастов, без минимума €/см³.
  - v4: акциз = 0 всегда.
  - API: реализовано под флагом в `estimate()`:
    - ФЛ: при `engine_type=="Электро"` → `duty_eur = price_eur * 0.15` (любой возраст).
    - ЮЛ: уже было 15%; подтверждено.
    - Акциз: if `v4_enabled and ELECTRO` → 0.

- **Гибриды (легковые, ЮЛ)**
  - v4: считать «как бензин» по таблицам (БД `DutyRate`).
  - API: под флагом гибриды ЮЛ перенаправлены в `_calc_duty(..., engine_type=BENZIN)`.

- **Не-авто (QUAD/SNOWMOBILE/MOTORCYCLE)**
  - v4: детальные таблицы по мощности/объёму/возрасту. План: внедрить после стабилизации EV/Hybrid, предпочтительно через БД (`DutyRate`) с новыми аудиториями.
  - API сейчас: упрощённые правила v2; без изменений (следующий этап интеграции).

- **Утильсбор (ФЛ, авто)**
  - v4: фиксированные суммы 3400/5200 RUB. В API достигается через коэффициенты к `Settings.util_base` и `UtilFee` фикстуры (без изменения кода). Проверить и при необходимости обновить seed с версированием.

- **НДС**
  - v4: только коммерческий режим (совпадает с API: `is_jur or is_personal_use is False`).

- **Таможенный сбор**
  - v4: по таблице ПП РФ №1637. В API совпадает (`CustomsFee`).

## Ссылки на код
- `api/calculator/services.py`:
  - `CustomsCalculator.estimate()` — ветки расчёта EV/Hybrid, чтение `V4_CALC_ENABLED`.
  - `_calc_duty()` — таблицы БД для авто.
  - `_calc_accise()` — акциз для авто (обнуление для EV под флагом выполняется на уровне `estimate()`).
- `api/calculator/models.py` — `DutyRate`, `CustomsFee`, `AcciseRate`, `UtilFee`, `Settings`.

## Тесты
