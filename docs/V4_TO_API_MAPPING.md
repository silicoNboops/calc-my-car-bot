# Customs Calculator V4 → API Mapping (Delta from V3)

Документ фиксирует отличия v4 от v3 и их реализацию в нашем API. Начиная с текущей версии, v4 логика активна по умолчанию (без фича‑флагов).

- **EV (легковые)**
  - v4: пошлина = 15% от стоимости для всех аудиторий и возрастов, без минимума €/см³ — реализовано.
  - v4: акциз = 0 — реализовано.

- **Гибриды (легковые)**
  - v4: считать «как бензин» по таблицам (БД `DutyRate`) — реализовано для ФЛ/ЮЛ; для последовательных/параллельных гибридов учтены особенности акциза.

- **Не‑авто (QUAD / SNOWMOBILE / MOTORCYCLE)** — реализовано через БД `DutyRate` с новыми аудиториями и HP‑юнитами.
  - Аудитории: `QUAD_PHYS|JUR`, `SNOWMOBILE_PHYS|JUR`, `MOTORCYCLE_PHYS|JUR`.
  - Возрастные группы: `under_3_years`, `over_5_years` (а также промежуточные у ЮЛ для мотоциклов при необходимости).
  - Новые единицы измерения ставок:
    - `EUR_HP` — фикс €/л.с. (например, ФЛ для quad/snowmobile от 3 лет и старше).
    - `PERCENT_HP` — % от цены с минимальным €/л.с. (например, ФЛ для quad/snowmobile до 3 лет; у ЮЛ — по группам).
  - Мотоциклы в v4 используют cc‑таблицы (`PERCENT`/`EUR_CC`) по возрасту и аудитории.
  - Расчёт в сервисе:
    - Для `PERCENT_HP`: `max(price_eur * rate_percent, hp * min_rate_eur_hp)`.
    - Для `EUR_HP`: `hp * rate_eur_hp`.
    - Для мотоциклов: аналогично авто — `%` с минимумом €/см³ или фикс €/см³.

- **Утильсбор**
  - ФЛ‑авто: детерминированный выбор коэффициента по строкам `UtilFee` с `max_cc IS NULL` — реализовано, чтобы исключить влияние шаблонных фикстур и порядка `NULL` в СУБД.
  - Non‑car (ФЛ/ЮЛ): сохранён согласованный подход на базе `Settings.util_base` с множителем 8.625 и v2‑коэффициентами, пока ставки утиля не перенесены в БД отдельно.

- **НДС**
  - v4: начисляется только в коммерческом режиме — как в API (`is_jur or not is_personal_use`).

- **Таможенный сбор**
  - v4: по таблице ПП РФ №1637 — реализовано в `CustomsFee`.
  - Версионированная фикстура V4 non‑car (`api/calculator/fixtures/customs_rates_v4_noncar_2025_08_17.json`) содержит блок `customs_fees`, идентичный таблице `CUSTOMS_FEES` в TS.

## Ссылки на код
- `api/calculator/services.py`:
  - `CustomsCalculator.estimate()` — маршрутизация по типу ТС (car/quad/snowmobile/motorcycle).
  - `_calc_duty_quad()` / `_calc_duty_snowmobile()` — HP‑таблицы `EUR_HP`/`PERCENT_HP`.
  - `_calc_duty_motorcycle()` — cc‑таблицы `PERCENT`/`EUR_CC`.
  - `_collect_hp_rows()` — выборка HP‑строк из БД.
  - `_calc_util()` — детерминированный выбор `UtilFee` и согласованный расчёт non‑car.
- `api/calculator/models.py` — модели: `DutyRate` (включая HP‑поля), `CustomsFee`, `AcciseRate`, `UtilFee`, `Settings`.

## Тесты
- `tests/unit/test_v4_non_car_duty.py` — целевые проверки quad / snowmobile / motorcycle по v4.
- `tests/unit/test_calculator.py` — общий набор: границы для авто, EV/Hybrid, smoke‑кейсы, часть v4 non‑car.
