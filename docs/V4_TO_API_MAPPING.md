# Customs Calculator V4 → API Mapping (Delta from V3)

Документ фиксирует отличия v4 от прежней реализации v3. Начиная с текущей версии, v4 логика активна по умолчанию (без фича-флагов).

- **EV (легковые)**
  - v4: пошлина = 15% от стоимости для всех аудиторий и возрастов, без минимума €/см³ — теперь это стандартное поведение API.
  - v4: акциз = 0 всегда.

- **Гибриды (легковые, ЮЛ)**
  - v4: считать «как бензин» по таблицам (БД `DutyRate`). Это поведение теперь всегда активно.

- **Не-авто (QUAD/SNOWMOBILE/MOTORCYCLE)**
  - v4: детальные таблицы по мощности/объёму/возрасту. План: внедрить через БД (`DutyRate`) с новыми аудиториями.
  - API сейчас: местами действуют упрощённые правила v2; задача на внедрение v4-таблиц остаётся открытой.

- **Утильсбор (ФЛ, авто)**
  - v4: фиксированные суммы 3400/5200 RUB. В API достигается через коэффициенты к `Settings.util_base` и `UtilFee` фикстуры (без изменения кода). При необходимости обновить seed с версионированием.

- **НДС**
  - v4: только коммерческий режим (совпадает с API: `is_jur or is_personal_use is False`).

- **Таможенный сбор**
  - v4: по таблице ПП РФ №1637. В API совпадает (`CustomsFee`).

## Ссылки на код
- `api/calculator/services.py`:
  - `CustomsCalculator.estimate()` — ветки расчёта EV/Hybrid реализованы как поведение по умолчанию (без флагов).
  - `_calc_duty()` — таблицы БД для авто.
  - `_calc_accise()` — акциз для авто (EV = 0).
- `api/calculator/models.py` — `DutyRate`, `CustomsFee`, `AcciseRate`, `UtilFee`, `Settings`.

## Тесты
- `tests/unit/test_v4_ev_rules.py` — проверка EV правил (duty=15% для ФЛ >3 лет; EV акциз=0 для ЮЛ/ФЛ) — без использования фича-флагов.
